## Why this matters

A bare `npx vitest run` in this repo currently reports **64 test files / 756
tests** when the real suite is **32 files / 378 tests**. The `include` glob in
`vitest.config.js` is unanchored (`'**/tests/**/*.vitest.js'`), so it also
matches test files inside `.claude/worktrees/agent-ace34737d41ee3c34/tests/` —
a git worktree checked out on the unmerged `feat/github-actions-ci` branch.
Every test runs twice, and a green run on `main` silently depends on whatever
state that unmerged branch is in: a diverged worktree would fail `main`'s
pre-push coverage gate for reasons unrelated to `main`. Any future agent
worktree under `.claude/` re-poisons the suite the same way.

Three smaller integrity gaps ride along: `npx knip` reports the load-bearing
`js/audio-converter.worker.js` as an "Unused file" (a standing false positive
that trains people to ignore knip output); `package.json` declares no Node
`engines` floor even though ESLint 9 / Vitest 3 / knip 5 all require modern
Node; and `package-lock.json` still records root `"version": "1.0.0"` while
`package.json` says `2.0.0`, so any future `npm install` produces a noisy
unrelated diff.

## Steps

### Step 1: Anchor the vitest test glob and exclude `.claude/`

In `vitest.config.js`:

1. Change the import line to also bring in defaults:
   `import { defineConfig, configDefaults } from 'vitest/config';`
2. Replace line 8 with an anchored glob and add an explicit exclude:

```js
    include: ['tests/**/*.vitest.js'],
    exclude: [...configDefaults.exclude, '**/.claude/**'],
```

(The anchored `include` alone stops the worktree pickup — worktree tests live
at `.claude/worktrees/<id>/tests/…`, which does not start with `tests/`. The
`exclude` entry is belt-and-braces against future config edits; spreading
`configDefaults.exclude` preserves the default `node_modules` exclusion.)

**Verify**: `npx vitest run 2>&1 | tail -6` → `Test Files  32 passed (32)` and
`Tests  378 passed (378)`, and no test path in the run output starts with
`.claude/`.

### Step 2: Declare the worker as a knip entry

In `knip.json`, change:

```json
"entry": ["js/main.js"],
```

to:

```json
"entry": ["js/main.js", "js/*.worker.js"],
```

**Verify**: `npx knip` → output contains no `Unused files` section (and no new
issue categories appear).

### Step 3: Add the Node engines floor

In `package.json`, add after the `"type": "module",` line:

```json
"engines": {
  "node": ">=20"
},
```

(Floor rationale: ESLint 9 requires `^18.18 || ^20.9 || >=21.1`, Vitest 3
requires `^18 || ^20 || >=22`; `>=20` is the simplest honest floor and covers
the happy-dom 20 bump coming in plan 004.)

**Verify**: `node -e "console.log(require('./package.json').engines.node)"` → `>=20`

### Step 4: Regenerate the lockfile root version

Run `npm install` (no arguments). This rewrites the lockfile's stale root
`"version": "1.0.0"` to `2.0.0` and records the new `engines` field.

**Verify**:
- `grep -c '"version": "2.0.0"' package-lock.json` → `2` (root + the `""` package entry)
- `git diff package-lock.json` shows only `version`/`engines` lines for the
  root entries — **no dependency version changes**. If dependency versions
  changed, see STOP conditions.

### Step 5: Run the full gate

**Verify**: `npm run test:coverage` → exit 0, all 378 tests pass, coverage
thresholds (85/80/70/85) met. Then `npm run lint` → exit 0, and
`npm run deps:check:prod` → exit 0.
