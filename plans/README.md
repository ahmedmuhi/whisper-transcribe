Net recommendation: trade the ~30 duplicates for ~12–15 gap-fills — same size,
strictly stronger. Not started; nothing here is broken.

## Findings considered and rejected

From the quick audit (2026-06-11, `3a83f9c`):

- **No CSP meta on `index.html` while Azure API keys persist in localStorage**
  (security, defense-in-depth): finding is real and was presented, but the
  operator chose not to plan it in this round. No XSS sink exists today
  (`textContent`/textarea throughout). Revisit if any HTML-injection surface
  is ever added. (Not rejected on merit — deferred by selection.)
- **`navigator.clipboard.writeText` called without a secure-context guard**
  (`js/ui.js:846`): the app already requires a secure context for
  `getUserMedia`, so the broken-clipboard environment cannot record.
- **Elapsed time derived from the rendered timer string**
  (`js/audio-handler.js:341-344`): up to ~1 s drift at the 10 s threshold and
  on pause/resume. Cosmetic; not worth churn in a well-tested path.
- **Stale saved input device id**: `OverconstrainedError` already handled by
  the fallback at `js/permission-manager.js:138-148`. By design.

New from the deep audit (2026-06-11, `50164c9`):

- **"Session model selector can diverge from stored credentials"**
  (`js/settings.js:120-139` vs `:682-690`): refuted on vetting — credentials
  are namespaced per model family and read keyed off the same selected model,
  so the pairing is always consistent; selecting an unconfigured model yields
  a clean, handled "API key is required" error. The session-only selector is
  explicitly intentional (code comment).
- **FSM `transitionTo` re-entrancy** (`js/recording-state-machine.js:87-112`):
  state handlers contain no real awaits (synchronous emits in async clothing),
  so the unguarded window is microtask-sized, and the transition table rejects
  illegal moves. A transition lock isn't worth the risk of masking tolerated
  reentrancy.
- **`_fetchWithRetry` can fall through and return `undefined`**: latent only —
  unreachable with current guards. Folded into plan 007 as the terminal-throw
  step rather than its own plan.
- **Visualizer sampler runs while tab hidden** (`js/visualization.js:104-110`):
  pausing sampling would leave a misleading flat gap in the waveform for time
  that *was* recorded; per-tick work is a 256-element RMS at 10 Hz. Not worth it.
- **Restrict the `?debug` logging switch** (`js/logger.js:53-63`): the toggle
  only raises console verbosity; vetting confirmed no credential ever reaches
  any log call (events carry `hasApiKey` booleans, not values). Documenting it
  (backlog DX item) beats restricting it; keep "no secrets in debug logs" as
  review discipline.
- **knip `plan/**` ignore is stale**: refuted — `plan/` exists (design docs).
  The worker-entry false positive was real and is fixed in plan 003.
- **Hidden-tab/battery and other micro-perf smells** (event-bus listener
  re-sort on subscribe, per-sample `setInt16` loop, textarea `+=` append):
  all measured against actual hot paths during vetting; none rise above
  micro-optimization at this app's scale.
