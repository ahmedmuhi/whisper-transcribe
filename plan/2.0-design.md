# Whisper Transcribe 2.0 — Design & Decision Log

This is the record of *what 2.0 became and why*. The original
`2.0-readiness-assessment*` documents (now in `plan/archive/`) framed 2.0 as a
**reliability** release. During the grill-me stress test that framing was
rejected: 2.0 is **interaction-and-experience-led**. Reliability items ride
along only where they serve the experience.

## The reframe

> 2.0 is about how the app *feels* to use, not a reliability checklist.

Everything below follows from that. Where a reliability fix made the experience
better (no UI freeze, no silent hang) it was included; where it didn't, it was
left for later.

## System principles (the non-negotiables)

1. **Proportional challenge.** Never a rote "are you sure?". Challenge the user
   only in proportion to what's actually at stake, and name what's at risk.
2. **Hit-target safety.** Decorative motion must *never* animate a button's
   geometry. (Scar tissue: earlier commits where a pulsing mic expanded its own
   click target and caused mis-clicks.) The recording "breath" is box-shadow /
   opacity only, guarded by `tests/mic-button-effects.vitest.js`.
3. **Single source of truth.** The controls render from one FSM state, not from
   a scatter of granular per-button events.
4. **No new runtime dependencies.** The whole Dynamic Island morph is native
   Web Animations API — no spring library, nothing to bundle.

## Key decisions

- **Dynamic Island, not a spring library.** The cluster morphs via FLIP
  (measure → reshape → animate the delta) using `element.animate()` with a
  settle ease (`cubic-bezier(0.22, 1, 0.36, 1)`, ~360ms). The container animates;
  the buttons never do. Overlapping morphs cancel the in-flight animation before
  measuring, so a fast state change can't seed the next morph from a mid-tween
  size. The recording breath pulses an **inset** ring so the cluster's
  `overflow: hidden` (which scopes the morph) doesn't clip it.
- **Success colour is teal, not mint.** Status needs an AA-contrast success hue
  distinct from the lavender accent and the red error. The first pass used a
  vivid mint that read as off-palette; 2.0 uses a desaturated teal
  (`#0E7A6B` light ≈ 5.2:1, `#5FD2C2` dark ≈ 9.5:1) that stays in the
  lavender/navy family. Enforced by `tests/status-tokens.vitest.js`.
- **Cut stays the hero; Restore is the safety net.** Cut lifts text to the
  clipboard (the common case); Restore brings back the last cut so an accidental
  clipboard clobber is recoverable. Clear is the explicit empty.
- **Discard is proportional.** Under ~10s → discard immediately (nothing to
  lose). Substantial recording → confirm once, by name.
- **Reduced motion is honoured in two layers, once each.** A single JS gate
  (`_prefersReducedMotion`) for the WAAPI morph + arrival, and one CSS
  `@media (prefers-reduced-motion)` block for keyframe-driven motion. Layout
  stays correct; only motion is removed.

## Process that paid off

Each slice ran **feature → adversarial-verify (independent skeptics) → /simplify
→ commit**. The verify passes caught real bugs the green suite missed (a retry
button vanishing in ERROR, a discard dead-end when `showModal` was unavailable,
the un-cancelled overlapping morph). The honest-test phase then deleted the
assertions that were only checking constants exist, and pinned the parts that
actually matter (island shape mapping, AA tokens, morph interruption).

## Deferred to 3.0

Out of scope for 2.0, recorded so the seams aren't painted over:

- Microsoft account sign-in (Entra ID)
- A Cosmos DB sync backend (the `TranscriptStore` seam is the hook for this)
- Topic mining across past transcripts
